# 全体像

## 目的
このソフトウェアの第一目標は、Discord と Codex CLI を仲介し、Discord を Codex の UI として使えるようにすることです。

## 長期方針
- チャットツール側は Discord 固定にせず、将来は他のチャットツールにも拡張できる構造にします。
- Agent 側は Codex CLI 固定にせず、将来は Claude Code など他の AI Agent にも拡張できる構造にします。
- 中核の業務ロジックは、チャット実装と Agent 実装から分離します。

## 対象範囲
この仕様は、チャットメッセージを受け取り、正規化し、Agent に渡し、応答を元のチャットへ返すまでを対象にします。

## 設計原則
- Adapter パターンで外部依存を隔離する
- チャットと Agent の間には共通の内部イベントモデルを置く
- セッションとメッセージ履歴は中核層で管理する
- まず単一プロセスで成立させ、その後に分散構成へ拡張できるようにする

## 初期スコープ
初期バージョンでは次を満たします。

- チャットツールは Discord のみ
- Agent は Codex CLI のみ
- 1 つの Discord 会話に対して 1 つの Agent セッションを対応付ける
- テキストメッセージの送受信を優先し、添付ファイルや高度な操作は後続対応とする

## 非対象
初期バージョンでは次を必須としません。

- 複数 Agent の同時協調
- 高度な権限制御
- 分散実行
- 複雑な監査基盤
